## üìù Documentation Standards

### File Organization
- **Main categories**: Use top-level folders (api/, deployment/, development/, etc.)
- **Sub-categories**: Use subfolders within main categories
- **File naming**: Use lowercase with hyphens (e.g., `admin-panel-guide.md`)

### Content Standards
- **Markdown**: All documentation uses `.md` extension
- **Plain text**: Supplementary files use `.txt` extension
- **Headers**: Use `# ## ###` hierarchy consistently
- **Links**: Use relative paths for internal documentation links
- **Code**: Use proper code blocks with language specification

### Maintenance
- **Updates**: Update documentation when code changes
- **Reviews**: Include documentation updates in code reviews
- **Accuracy**: Keep all examples and commands current
